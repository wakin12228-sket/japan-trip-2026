## ADR-001：採用一桌三櫃工作流

**日期：** 2026-06-20  
**狀態：** 已採用

**背景：**  
需要在不同電腦間切換工作，並解決 AI 長對話上下文溢出的問題。

**決策：**  
- 工作目錄放 Google Drive 同步
- 程式碼版本控制用 GitHub
- 動態資料用 Firebase
- 進度與決策用 Obsidian（Markdown）
- 用 cloud.md 作為 AI 快速恢復上下文的入口

**後果：**  
- 每次收工需更新 cloud.md 與 Obsidian
- 新對話開工前先讀 cloud.md，可保持 AI 最佳狀態

## ADR-002：靜態 JSON、免登入、缺漏留空

**日期：** 2026-06-20  
**狀態：** 已採用

**背景：**  
個人家族行程，主要需求是手機/電腦查看；改動頻率低，不需即時同步或登入。

**決策：**  
- 資料存 `src/data/itinerary.json`，前端 Vanilla JS 渲染
- 不用 Firebase Auth / Firestore（v1）
- 只匯入使用者提供的 Word 內容；Day 4、6–9 及 Day 5 上午留空，AI 不寫 placeholder 文案
- 部署預設 GitHub Pages（`src/` 為根目錄）

**後果：**  
- 改行程 = AI 改 JSON + git push，約 1–3 分鐘上線
- 本地預覽需簡易 HTTP server（fetch JSON）

## ADR-003：頁首角色用單一 family.png

**日期：** 2026-06-21  
**狀態：** 已採用

**背景：**  
原先布布、一二各一張分開的圖；使用者提供全家福 `family.png`。

**決策：**  
`meta.characters` 改為單一項目，指向 `images/characters/family.png`；頁首 CSS 改為寬幅顯示。

**後果：**  
舊的 characters 分圖已從 repo 移除。
