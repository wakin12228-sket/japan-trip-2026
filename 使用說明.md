# 一桌三櫃 — 使用說明

> AI 專案管理工作流懶人包（Cursor 版）

---

## 資料夾結構

```
一桌3櫃/
├── cloud.md                 ← 專案藍圖（AI 開工必讀）
├── .cursor/rules/           ← Cursor 自動規則（含收工流程）
├── src/                     ← 程式碼放這裡
├── obsidian/                ← 筆記（可加入 Obsidian vault）
│   ├── 進度.md
│   ├── 架構決策.md
│   └── 踩坑紀錄.md
├── firebase/                ← Firebase 設定
└── scripts/shougon.sh       ← 收工腳本（可選）
```

---

## 第一次設定（只需做一次）

### 1. Google Drive 同步

把整個 `一桌3櫃` 資料夾移到 Google Drive 同步目錄，或在 Drive 裡建立後用 Cursor 開啟。

### 2. GitHub

```bash
cd "一桌3櫃"
git init
git remote add origin https://github.com/你的帳號/你的repo.git
git add .
git commit -m "初始化一桌三櫃工作流"
git push -u origin main
```

### 3. Firebase

1. 前往 [Firebase Console](https://console.firebase.google.com/) 建立專案
2. 把專案 ID 填入 `firebase/.firebaserc`
3. 安裝 CLI：`npm install -g firebase-tools`
4. 登入並初始化：`firebase login` → `cd firebase && firebase init`

### 4. Obsidian（選用）

在 Obsidian 開啟 `obsidian/` 資料夾作為 vault，或直接在本 repo 裡編輯 `.md` 檔。

## 日常使用

### 開工（新對話）

在 Cursor 新開 Chat / Agent，貼上：

```
請先讀取 cloud.md 和 obsidian/ 資料夾內的所有筆記，
掌握專案進度後接續工作。
```

Cursor Rules 已設定好，通常直接說「開工」或「接續」也可以。

### 收工

在 Cursor 說：

```
收工
```

AI 會更新 cloud.md、Obsidian 筆記，並準備 git commit。  
確認後再 push。

或用終端機腳本（需先 `chmod +x scripts/shougon.sh`）：

```bash
./scripts/shougon.sh "收工：完成登入頁面"
```

### 長對話提醒

對話太長時，AI 會建議收工。開新對話 + 讀 cloud.md = 保持最佳 AI 狀態。
